### active-memory-system ###
为 nanobot 实现一套记忆管理系统，包含异步上下文压缩（QA 对提取）、长期记忆自动提取、事件记忆关键字召回工具，替换现有的纯文本 Summarizer。基于 nanobot 的轻量级文件系统 + SQLite 存储方案，无需 MySQL 和多租户隔离。

## 实施计划：Active Memory 记忆管理系统（nanobot 适配版）

### 一、整体设计概述

在现有 AgentLoop 框架基础上，**替换现有 `Summarizer`**，新增一套记忆管理系统，核心能力包括：

1. **异步上下文压缩**：当 LLM 返回的 `prompt_tokens` 达到阈值时，异步触发压缩，将对话压缩为结构化 QA 对，不阻塞主流程
2. **长期记忆自动提取**：从压缩内容中提取用户偏好/规范类信息，自动 append 到 `workspace/memory/MEMORY.md`
3. **事件记忆（QA 对）**：从压缩内容中提取已完成事件的 (Q, A) 对，存储到 SQLite，通过关键字索引支持检索
4. **记忆召回工具**：注册 `memory_recall` 工具（继承 `Tool` 基类），供 Agent 在对话中主动召回历史事件记忆

### 二、与现有系统的关系

| 现有模块 | 变更方式 | 说明 |
|----------|----------|------|
| `agent/summarizer.py` | **替换** → `agent/compressor.py` | 压缩产出 QA 对 + 长期记忆，而非纯文本摘要 |
| `agent/memory.py` (MemoryStore) | **扩展** | 增加 `append_long_term` 方法，支持自动提取的记忆写入 |
| `agent/loop.py` (AgentLoop) | **修改** | `_maybe_trigger_summarization` 改为调用新压缩器 |
| `agent/context.py` (ContextBuilder) | **不变** | `build_messages` 中 `summary` 参数接口不变，内容从纯文本变为 QA 拼接摘要 |
| `session/manager.py` (Session) | **扩展** | Session 增加 `compression_state` 字段 |
| `config/schema.py` | **扩展** | `AgentDefaults` 增加 `compress_model` 配置 |

#### 步骤 1：数据模型与存储 (`nanobot/agent/compressor.py` 内)

**SQLite 表结构**：

```sql
-- 事件记忆表
CREATE TABLE IF NOT EXISTS event_memories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_key TEXT NOT NULL,
    question TEXT NOT NULL,
    conclusion TEXT NOT NULL,
    created_at TEXT NOT NULL
);

-- 事件记忆关键字索引表
CREATE TABLE IF NOT EXISTS event_keywords (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    event_id INTEGER NOT NULL,
    keyword TEXT NOT NULL,
    FOREIGN KEY (event_id) REFERENCES event_memories(id)
);

CREATE INDEX IF NOT EXISTS idx_event_keywords_keyword ON event_keywords(keyword);

-- 压缩状态表
CREATE TABLE IF NOT EXISTS compression_states (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_key TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'pending',
    compressed_event_count INTEGER NOT NULL DEFAULT 0,
    qa_pairs_json TEXT,
    created_at TEXT NOT NULL,
    completed_at TEXT
);

CREATE INDEX IF NOT EXISTS idx_compression_session ON compression_states(session_key);
```

**业务数据模型（dataclass）**：

```python
from dataclasses import dataclass

@dataclass(frozen=True, slots=True)
class LongTermMemoryItem:
    category: str    # preference / convention / lesson
    content: str

@dataclass(frozen=True, slots=True)
class EventMemoryItem:
    question: str
    conclusion: str
    keywords: list[str]

@dataclass(slots=True)
class CompressionResult:
    qa_pairs: list[EventMemoryItem]
    long_term_memories: list[LongTermMemoryItem]
    compressed_event_count: int

    @property
    def reconstructed_summary(self) -> str:
        """从 QA 对直接拼接生成摘要，不需要额外的 LLM 调用"""
        lines = []
        for i, qa in enumerate(self.qa_pairs, 1):
            lines.append(f"[Event {i}] Q: {qa.question}\nA: {qa.conclusion}")
        return "\n\n".join(lines)
```

#### 步骤 2：上下文压缩器 (`nanobot/agent/compressor.py`)

**替换现有 `summarizer.py`**，核心职责：

- 异步后台执行（fire-and-forget，与现有 Summarizer 相同模式）
- 调用 LLM 将对话压缩为 QA 对 + 提取长期记忆
- 将 QA 对存入 SQLite，长期记忆 append 到 MEMORY.md
- 更新 Session 的 summary + compression_state

**关键接口**（与 Summarizer 保持一致）：

```python
class Compressor:
    def __init__(self, provider: LLMProvider, model: str, workspace: Path):
        ...

    def fire_and_forget(
        self,
        session: Session,
        session_manager: SessionManager,
        messages_snapshot: list[dict[str, Any]],
        previous_summary: str,
        min_keep: int,
    ) -> None:
        """接口与原 Summarizer.fire_and_forget 完全一致"""
        ...
```

**压缩 Prompt 要点**：
- 输入为完整的对话上下文（messages），要求 LLM 输出 JSON
- 输出包含：`qa_pairs`（已完成事件的 Q/A 对 + keywords）、`long_term_memories`（用户偏好/规范）、`compressed_event_count`
- 关键指令：尚未完成的事件（问题提出但无结论）不压缩
- keywords 应包含关键实体和主题标签

**压缩完成后的处理流程**：

```
_do_compress():
    1. 调用 LLM 压缩 → CompressionResult
    2. 存储事件记忆 + 关键字索引 → SQLite
    3. 长期记忆 append → workspace/memory/MEMORY.md
    4. 安全裁剪 session.messages（按完整 event 边界）
    5. session.summary = reconstructed_summary（复用现有 summary 字段）
    6. session_manager.save(session)
```

### 五、裁剪边界安全性

裁剪必须在**完整的 event 边界**上进行，不能在 `assistant(tool_calls)` 和 `tool(response)` 之间截断。

```python
def trim_compressed_events(messages: list[dict], compressed_event_count: int) -> list[dict]:
    """
    安全裁剪 messages，确保裁剪边界在完整的 event 边界上。
    
    一个完整的 event：user → 0~N 轮 (assistant+tool_calls → tool responses) → assistant final
    """
    event_starts = [i for i, msg in enumerate(messages) if msg.get("role") == "user"]
    
    if compressed_event_count >= len(event_starts):
        return []
    
    split_point = event_starts[compressed_event_count]
    remaining = messages[split_point:]
    
    # 安全验证：确保不以 tool response 开头
    while remaining and remaining[0].get("role") == "tool":
        remaining.pop(0)
    
    return remaining
```

### 六、完整数据流

```
用户消息
    │
    ▼
[AgentLoop._process_message_inner]
    │
    ├── session.get_history()
    │   └── 如果有 compression_state，自动裁剪已压缩 events
    │
    ├── context.build_messages(summary=session.summary)
    │   └── summary 内容 = reconstructed_summary（QA 对拼接的结构化摘要）
    │       + MEMORY.md 中的长期记忆（通过现有 memory context 机制自动注入）
    │
    ├── LLM 循环（tool calls / final response）
    │   └── agent 可调用 memory_recall 工具主动召回事件记忆
    │
    ├── 保存到 Session
    │
    └── _maybe_trigger_summarization(session, last_response)
        │
        └── 如果 prompt_tokens >= threshold:
            │
            └── compressor.fire_and_forget() ──→ 异步后台执行：
                │
                ├── [1] 调用 LLM 压缩 → CompressionResult
                │       ├── qa_pairs: [{question, conclusion, keywords}, ...]

                ├── [2] 存储事件记忆 + 关键字索引 → SQLite (memory.db)
                │

                ├── [4] 安全裁剪 session.messages（按 event 边界）

                ├── [5] session.summary = reconstructed_summary

```

### 七、关键设计决策

| 决策点 | 方案 | 理由 |
|--------|------|------|
| 存储方案 | SQLite + 文件系统 | 保持 nanobot 轻量级定位，无需外部数据库依赖 |
| 替换 vs 并存 | 替换 Summarizer | 功能完全覆盖，避免两套系统并存的复杂度 |
| 长期记忆存储 | 复用 MEMORY.md | nanobot 已有此机制，agent 也已经会读写它 |
| 原始对话保存 | 不需要额外表 | 现有 JSONL Session 已存完整对话 |
| 多租户隔离 | 不需要 | nanobot 是单用户个人助手，session_key 已提供会话隔离 |
| 长期记忆去重 | MVP 阶段不做 | 个人场景记忆量小，后续可定期让 agent 整理 MEMORY.md |
| 触发机制 | 复用现有 prompt_tokens 判断 | 比字符估算更精确，且已经过验证 |
| 压缩器接口 | 与 Summarizer 保持一致 | `fire_and_forget` 签名不变，AgentLoop 改动最小 |
| 事件记忆检索 | SQLite 关键字索引 | 个人场景事件量 <1000 条，完全够用 |
| summary 字段复用 | 复用 Session.summary | 不新增字段，reconstructed_summary 直接存入 summary |
