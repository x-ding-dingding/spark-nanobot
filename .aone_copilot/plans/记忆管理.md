### active-memory-system ###
为 nanobot 实现一套记忆管理系统，包含异步上下文压缩（QA 对提取）、长期记忆自动提取、事件记忆关键字召回工具，替换现有的纯文本 Summarizer。基于 nanobot 的轻量级文件系统 + SQLite（aiosqlite）存储方案。

## 实施计划：Active Memory 记忆管理系统（nanobot 适配版）

### 一、整体设计概述

在现有 AgentLoop 框架基础上，**替换现有 `Summarizer`**，新增一套记忆管理系统，核心能力包括：

1. **异步上下文压缩**：当 LLM 返回的 `prompt_tokens` 达到阈值时，异步触发压缩，将对话压缩为结构化 QA 对，不阻塞主流程
2. **长期记忆自动提取**：从压缩内容中提取用户偏好/规范类信息，自动 append 到 `workspace/memory/MEMORY.md`（已在 `build_system_prompt` 中通过 `MemoryStore.get_memory_context()` 自动注入 system prompt，无需额外操作）
3. **事件记忆（QA 对）**：从压缩内容中提取已完成事件的 (Q, A) 对，存储到 SQLite，通过关键字索引支持检索
4. **记忆召回工具**：注册 `memory_recall` 工具（继承 `Tool` 基类），供 Agent 在对话中主动召回历史事件记忆

### 二、与现有系统的关系

| 现有模块 | 变更方式 | 说明 |
|----------|----------|------|
| `agent/summarizer.py` | **替换** → `agent/compressor.py` | 压缩产出 QA 对 + 长期记忆，而非纯文本摘要 |
| `agent/memory.py` (MemoryStore) | **扩展** | 增加 `append_long_term` 方法，支持自动提取的记忆写入 |
| `agent/loop.py` (AgentLoop) | **修改** | `_maybe_trigger_summarization` 改为调用新压缩器 |
| `agent/context.py` (ContextBuilder) | **不变** | `build_messages` 中 `summary` 参数接口不变，内容从纯文本变为 QA 拼接摘要 |
| `session/manager.py` (Session) | **不变** | 复用现有 `summary` 字段存 `reconstructed_summary`，无需新增字段 |
| `config/schema.py` | **扩展** | `AgentDefaults` 增加 `compress_model` 配置 |

#### 步骤 1：数据模型与存储

**SQLite 表结构**（通过 aiosqlite 管理）：

```sql
-- 事件记忆表
CREATE TABLE IF NOT EXISTS event_memories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_key TEXT NOT NULL,
    question TEXT NOT NULL,
    conclusion TEXT NOT NULL,
    created_at TEXT NOT NULL
);

-- 事件记忆关键字索引表（每个 event 一条记录，keyword 存复合键）
-- 每个事件只有一个 composite_key，不再是多关键词
CREATE TABLE IF NOT EXISTS event_keywords (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    event_id INTEGER NOT NULL,
    keyword TEXT NOT NULL,          -- 复合键，如 'nanobot-配置迁移-项目目录'
    FOREIGN KEY (event_id) REFERENCES event_memories(id)
);

CREATE INDEX IF NOT EXISTS idx_event_keywords_keyword ON event_keywords(keyword);
```

> 注意：不再需要 `compression_states` 表。裁剪逻辑是固定的"保留最近 3 轮"，不依赖压缩状态。

**业务数据模型（dataclass）**：

```python
from dataclasses import dataclass

@dataclass(frozen=True, slots=True)
class LongTermMemoryItem:
    category: str    # preference / convention / lesson
    content: str

@dataclass(frozen=True, slots=True)
class EventMemoryItem:
    question: str
    conclusion: str
    composite_key: str  # 复合键，如 "nanobot-配置迁移-项目目录"

@dataclass(slots=True)
class CompressionResult:
    qa_pairs: list[EventMemoryItem]
    long_term_memories: list[LongTermMemoryItem]

    @property
    def reconstructed_summary(self) -> str:
        """从 QA 对直接拼接生成摘要，不需要额外的 LLM 调用"""
        lines = []
        for i, qa in enumerate(self.qa_pairs, 1):
            lines.append(f"[Event {i}] Q: {qa.question}\nA: {qa.conclusion}")
        return "\n\n".join(lines)
```

#### 步骤 1.5：微压缩 (`nanobot/agent/mini_compact.py`)

**纯规则、零 LLM 调用**的上下文瘦身，在每次 LLM 请求前由 `build_messages` 流程调用。与 Compressor 配合形成两层压缩体系：

| 层级 | 模块 | 触发时机 | 成本 |
|------|------|----------|------|
| 第一层 — 微压缩 | `mini_compact` | 每次 LLM 请求前 | 零（纯规则） |
| 第二层 — 深度压缩 | `Compressor` | prompt_tokens 超阈值时 | 一次 LLM 调用 |

**三层瘦身策略**：

```python
# 配置常量
RECENT_TURNS_TO_KEEP = 3          # 保留最近 3 轮完整 tool result
LARGE_OUTPUT_THRESHOLD = 10_000   # 单条 tool result 超过此字符数则截断
LARGE_OUTPUT_HEAD = 4_000         # 截断时保留头部字符数
LARGE_OUTPUT_TAIL = 2_000         # 截断时保留尾部字符数
```

1. **旧 tool result 替换**：只保留最近 N 轮的完整 tool result，更早的替换为 `[tool result omitted for brevity]`
2. **大输出截断**：近 N 轮中，单条 tool result 超过 10,000 字符时，只保留前 4,000 + 后 2,000 字符，中间用 `... [X chars omitted] ...` 替代
3. **图片/文档替换**：所有 `image_url` → `[image]`，`document` → `[document]`

**核心函数**：

```python
def mini_compact(history: list[dict]) -> list[dict]:
    """返回新列表（不修改原始 history）"""
    compacted = copy.deepcopy(history)
    
    # 1. 找到所有 tool-call turn 的边界
    turn_boundaries = _find_tool_turn_boundaries(compacted)
    recent_cutoff = len(turn_boundaries) - RECENT_TURNS_TO_KEEP
    
    for turn_index, (assistant_idx, tool_indices) in enumerate(turn_boundaries):
        is_old_turn = turn_index < recent_cutoff
        for tool_idx in tool_indices:
            msg = compacted[tool_idx]
            if is_old_turn:
                msg["content"] = "[tool result omitted for brevity]"  # 旧轮次直接替换
            elif len(msg.get("content", "")) > LARGE_OUTPUT_THRESHOLD:
                msg["content"] = _truncate_head_tail(content)         # 近轮次截断大输出
    
    # 2. 替换所有图片/文档 content blocks
    for msg in compacted:
        if isinstance(msg.get("content"), list):
            msg["content"] = _compact_content_blocks(content)
    
    return compacted
```

**"tool turn" 的定义**：一个带 `tool_calls` 的 assistant 消息 + 紧随其后的一个或多个 `role=tool` 消息。`_find_tool_turn_boundaries()` 从头到尾扫描，收集所有 `(assistant_idx, [tool_indices])` 对。

#### 步骤 2：上下文压缩器 (`nanobot/agent/compressor.py`)

**替换现有 `summarizer.py`**。

**核心设计变更**：

1. **不使用独立 Prompt**：在当前对话的 messages 快照上追加一条 user message 作为压缩指令，复用 LLM 的 KV cache
2. **全部 messages 送 LLM**：不做前后切割，让 LLM 自行判断哪些事件已完成、哪些首尾不完整应跳过
3. **裁剪与压缩解耦**：裁剪逻辑固定为"保留最近 3 轮"，不依赖 LLM 返回的任何字段

**关键接口**（与 Summarizer 保持一致）：

```python
class Compressor:
    RECENT_TURNS_TO_KEEP = 3

    def __init__(self, provider: LLMProvider, model: str, workspace: Path):
        self.provider = provider
        self.model = model
        self.workspace = workspace
        self._db_path = get_data_path() / "memory.db"
        self._db_initialized = False
        # 注意：MemoryStore 在 _append_long_term_memories() 中懒初始化，不在 __init__ 中创建

    def fire_and_forget(
        self,
        session: Session,
        session_manager: SessionManager,
        messages_snapshot: list[dict[str, Any]],
        previous_summary: str,
        min_keep: int,  # 保留接口兼容，内部不使用（用 turn-based 裁剪替代）
    ) -> None:
        task = asyncio.create_task(
            self._do_compress(session, session_manager, messages_snapshot, previous_summary)
        )
        task.add_done_callback(self._on_task_done)
```

**`_do_compress` 核心流程**：

```python
async def _do_compress(self, session, session_manager, messages_snapshot, previous_summary, min_keep):
    try:
        # 1. 在 messages 快照上追加压缩指令（复用 KV cache）
        compress_messages = copy.deepcopy(messages_snapshot)
        if previous_summary:
            instruction = COMPRESSION_INSTRUCTION_WITH_PREVIOUS.format(previous_summary=previous_summary)
        else:
            instruction = COMPRESSION_INSTRUCTION
        compress_messages.append({"role": "user", "content": instruction})

        # 2. 调用 LLM
        response = await self.provider.chat(
            messages=compress_messages, tools=None,
            model=self.model, max_tokens=2048, temperature=0.3,
        )

        # 3. 解析 JSON 响应（带 fallback）
        result = self._parse_compression_response(response.content)
        if result is None:
            logger.warning("Compression: failed to parse LLM response, skipping")
            return

        # 4. 存储事件记忆 + 关键字索引 → SQLite (aiosqlite)
        await self._save_event_memories(session.key, result.qa_pairs)

        # 5. 长期记忆 append → MEMORY.md
        if result.long_term_memories:
            self.memory.append_long_term(result.long_term_memories)

        # 6. summary 替换：旧 QA 已存入事件记忆，可通过召回获取，无需累积
        if result.qa_pairs:
            session.summary = result.reconstructed_summary
        # 若本次未提取到 qa_pairs，保持 previous_summary 不变

        # 7. 安全裁剪 session.messages：保留最近 3 轮
        #    从 session.messages 的【当前状态】取尾部（防竞态，压缩期间可能有新消息）
        kept = self._keep_recent_turns(session.messages, turns_to_keep=3)
        session.messages = kept

        # 8. 保存
        session.summary_in_progress = False
        session_manager.save(session)

    except Exception as exc:
        logger.warning(f"Compression error for {session.key}: {exc}")
    finally:
        session.summary_in_progress = False
```

**安全裁剪逻辑**：

```python
@staticmethod
def _keep_recent_turns(messages: list[dict], turns_to_keep: int = 3) -> list[dict]:
    """保留最近 N 轮完整 messages。
    
    一轮 = user message → 0~N 个 (assistant+tool_calls → tool responses) → assistant final response
    
    从尾部往前数 turns_to_keep 个 role=user 的位置，保留从该位置到末尾的所有 messages。
    裁剪点天然在 user message 边界上，不会截断 tool_calls/tool response 对。
    """
    user_indices = [i for i, msg in enumerate(messages) if msg.get("role") == "user"]
    
    if len(user_indices) <= turns_to_keep:
        return list(messages)  # 不够 N 轮，全部保留
    
    keep_from = user_indices[-turns_to_keep]
    return list(messages[keep_from:])
```

**压缩指令（追加为 user message）**：

```python
COMPRESSION_INSTRUCTION = """Please analyze our conversation above and extract the following in JSON format:

{
  "qa_pairs": [
    {
      "question": "concise question",
      "conclusion": "concise conclusion/answer",
      "composite_key": "entity1-entity2-entity3"
    }
  ],
  "long_term_memories": [
    {
      "category": "requirement|correction|constraint",
      "content": "memory content"
    }
  ]
}

Rules:
1. qa_pairs: Extract completed events/tasks as Q&A pairs. Skip any incomplete events.
2. composite_key: A single hyphen-joined string of 2-4 SPECIFIC named entities that uniquely
   identify this event (project names, feature names, tool names, person names, unique identifiers).
   NEVER use generic status words ("未开始", "进行中", "完成"), generic action words ("任务进度",
   "确认", "更新"), or adjectives. The composite_key must uniquely identify THIS event.
   Example: "nanobot-配置迁移-项目目录" NOT "任务-进度-未开始"
3. long_term_memories: ONLY extract when the user EXPLICITLY states a requirement, correction,
   or complaint in their own words.
   Valid triggers:
   - User gives a future instruction: "以后/下次/记住/不要再..."
   - User corrects a mistake: "这个不对", "你搞错了", "应该是..."
   - User states a clear constraint: "我不喜欢X", "必须用Y方式"
   Do NOT extract:
   - Inferred habits or preferences from conversation tone or style
   - Content from the system prompt, agent persona, or SOUL.md
   - One-time behaviors that may not be recurring preferences
   If in doubt, leave long_term_memories as an empty array [].
4. Only output valid JSON, no other text."""
```

**JSON 解析鲁棒性**：

```python
def _parse_compression_response(self, content: str) -> CompressionResult | None:
    """解析 LLM 返回的 JSON，带 fallback。"""
    if not content:
        return None
    try:
        # 尝试直接解析
        data = json.loads(content)
    except json.JSONDecodeError:
        # Fallback: 尝试从 markdown code block 中提取
        import re
        match = re.search(r'```(?:json)?\s*(\{.*?\})\s*```', content, re.DOTALL)
        if match:
            try:
                data = json.loads(match.group(1))
            except json.JSONDecodeError:
                return None
        else:
            return None

    qa_pairs = [
        EventMemoryItem(
            question=qa.get("question", ""),
            conclusion=qa.get("conclusion", ""),
            composite_key=qa.get("composite_key", "").strip().lower(),
        )
        for qa in data.get("qa_pairs", [])
        if qa.get("question") and qa.get("conclusion") and qa.get("composite_key")
    ]
    long_term = [
        LongTermMemoryItem(
            category=m.get("category", "general"),
            content=m.get("content", ""),
        )
        for m in data.get("long_term_memories", [])
        if m.get("content")
    ]
    return CompressionResult(qa_pairs=qa_pairs, long_term_memories=long_term)
```

**Compressor 查询 API**（供 MemoryRecallTool 调用）：

```python
async def get_all_composite_keys(self) -> list[str]:
    """返回所有已存储的 composite_key。
    
    MemoryRecallTool 先调用此方法获取所有 key，
    然后通过独立 LLM 调用选择相关 key。
    """
    await self._ensure_db()
    async with aiosqlite.connect(self._db_path) as db:
        async with db.execute(
            "SELECT DISTINCT keyword FROM event_keywords ORDER BY keyword"
        ) as cursor:
            rows = await cursor.fetchall()
    return [row[0] for row in rows]

async def search_by_composite_keys(
    self, composite_keys: list[str], top_k: int = 5
) -> list[dict[str, Any]]:
    """按 composite_key 精确匹配查询事件记忆。
    
    Args:
        composite_keys: LLM 从 get_all_composite_keys() 中选择的 key 列表
        top_k: 最多返回结果数
    
    Returns:
        按创建时间倒序的事件列表，包含 id, question, conclusion, composite_key, created_at
    """
    await self._ensure_db()
    if not composite_keys:
        return []
    
    normalized = [key.strip().lower() for key in composite_keys if key.strip()]
    placeholders = ",".join("?" for _ in normalized)
    
    query = f"""
        SELECT em.id, em.session_key, em.question, em.conclusion, em.created_at,
               ek.keyword AS composite_key
        FROM event_memories em
        JOIN event_keywords ek ON ek.event_id = em.id
        WHERE ek.keyword IN ({placeholders})
        ORDER BY em.created_at DESC
        LIMIT ?
    """
    
    async with aiosqlite.connect(self._db_path) as db:
        db.row_factory = aiosqlite.Row
        async with db.execute(query, (*normalized, top_k)) as cursor:
            rows = await cursor.fetchall()
    
    return [
        {
            "id": row["id"],
            "question": row["question"],
            "conclusion": row["conclusion"],
            "composite_key": row["composite_key"],
            "created_at": row["created_at"],
        }
        for row in rows
    ]
```

#### 步骤 3：记忆召回工具 (`nanobot/agent/tools/memory_recall.py`)

继承 nanobot 的 `Tool` 基类，内部两步走召回，完全不污染主对话上下文：

```python
_CONTEXT_TURNS = 5  # 独立 LLM 调用时传入的最近对话轮数

_KEY_SELECTION_SYSTEM = (
    "You are a memory retrieval assistant. "
    "Given a list of memory keys and recent conversation context, "
    "select the keys most relevant to the user's current query. "
    "Return ONLY the selected keys, one per line, no explanation."
)

class MemoryRecallTool(Tool):
    """两步走召回：
    1. 获取所有复合键列表 → 独立 LLM 调用（传入最近几轮对话上下文）选择相关复合键
    2. 用选中的复合键精确查 SQLite → 返回 event 内容作为 tool result
    """

    def __init__(self, compressor: "Compressor", provider: "LLMProvider") -> None:
        self._compressor = compressor
        self._provider = provider
        self._session: "Session | None" = None  # 通过 set_session() 注入

    def set_session(self, session: "Session") -> None:
        """在每次消息处理前由 AgentLoop 调用，更新当前 session 引用。"""
        self._session = session

    @property
    def name(self) -> str:
        return "memory_recall"

    @property
    def parameters(self) -> dict:
        return {
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",
                    "description": "描述你想召回的内容，用自然语言说明当前需要查找什么历史事件"
                },
                "top_k": {
                    "type": "integer",
                    "description": "Max number of memories to return (default 5)"
                }
            },
            "required": ["query"]
        }

    async def execute(self, **kwargs) -> str:
        query = kwargs.get("query", "").strip()
        top_k = int(kwargs.get("top_k", 5))

        if not query:
            return "Please describe what you want to recall."
        if self._session is None:
            return "Memory recall is not available yet (session not initialized)."

        # Step 1: 获取所有复合键
        all_keys = await self._compressor.get_all_composite_keys()
        if not all_keys:
            return "No memories available yet — memory index is empty."

        # Step 2: 独立 LLM 调用选择相关复合键
        selected_keys = await self._select_keys_via_llm(all_keys, query)
        if not selected_keys:
            return f"No relevant memories found for: {query}"

        # Step 3: 精确查 SQLite
        results = await self._compressor.search_by_composite_keys(selected_keys, top_k=top_k)
        if not results:
            return f"No memories found for selected keys: {', '.join(selected_keys)}"

        return self._format_results(results)

    async def _select_keys_via_llm(self, all_keys: list[str], query: str) -> list[str]:
        """独立 LLM 调用选择相关复合键。
        
        关键设计：使用子串匹配而非精确行匹配来解析 LLM 输出，
        因为 LLM 可能输出 "nanobot-配置迁移 是最相关的" 而非纯键名。
        """
        recent_turns = self._get_recent_dialogue_turns(_CONTEXT_TURNS)
        keys_block = "\n".join(f"- {key}" for key in all_keys)
        
        selection_prompt = (
            f"User query: {query}\n\n"
            f"Available memory keys:\n{keys_block}\n\n"
            "Select the keys most relevant to the query above. "
            "Return only the selected keys, one per line. "
            "If none are relevant, return an empty response."
        )

        messages = (
            [{"role": "system", "content": _KEY_SELECTION_SYSTEM}]
            + recent_turns
            + [{"role": "user", "content": selection_prompt}]
        )

        response = await self._provider.chat(
            messages=messages, tools=None,
            model=self._compressor.model, max_tokens=256, temperature=0.0,
        )

        raw = (response.content or "").strip()
        if not raw:
            return []

        # 子串匹配：检查每个已知 key 是否出现在 LLM 输出中
        raw_lower = raw.lower()
        selected = []
        for key in all_keys:
            key_lower = key.lower()
            if key_lower in raw_lower:
                selected.append(key_lower)
        return selected

    def _get_recent_dialogue_turns(self, max_turns: int) -> list[dict]:
        """从 session 取最近 N 轮 user/assistant 消息，跳过 tool_calls/tool result。"""
        messages = self._session.messages
        dialogue = []
        for msg in reversed(messages):
            role = msg.get("role", "")
            if role == "tool":
                continue
            if role == "assistant" and msg.get("tool_calls") and not msg.get("content"):
                continue
            if role in ("user", "assistant"):
                dialogue.append({"role": role, "content": msg.get("content", "")})
            if len(dialogue) >= max_turns * 2:
                break
        return list(reversed(dialogue))
```

**关键设计点**：
- `MemoryRecallTool` 通过 `set_session()` 方法在每次消息处理前注入 session（而非构造时传入）
- 独立 LLM 调用的 messages **不进入主对话上下文**，主上下文只注入最终召回结果
- 独立调用的 system 极简（只说明任务），不带 SOUL.md/TOOLS.md 等 agent persona
- 对外暴露的参数从 `keywords: list` 改为 `query: str`（自然语言描述），模型不需要自己想关键词
- **LLM 输出解析使用子串匹配**：遍历所有已知 key，检查是否出现在 LLM 输出中，比精确行匹配更健壮

在 `AgentLoop._register_default_tools()` 中注册，传入 `compressor` 和 `provider`。
在 `AgentLoop._process_message_inner()` 开头调用 `memory_recall_tool.set_session(session)`。

**4.3 `agent/loop.py` — Tool Result 截断 (`_truncate_tool_result`)**

当 tool 结果存入 session 持久化时，对超长输出进行 head+tail 截断，避免 session 文件膨胀：

```python
def _truncate_tool_result(result: str, max_chars: int = 2000) -> str:
    """完整结果用于当前轮 LLM 上下文，截断版本存入 session 持久化。
    保留前 70% + 后 30% 字符，确保头部上下文和尾部状态/报错都能保留。
    """
    if len(result) <= max_chars:
        return result
    head_chars = int(max_chars * 0.7)  # ~1400
    tail_chars = max_chars - head_chars  # ~600
    return f"{result[:head_chars]}\n... [{omitted} chars omitted] ...\n{result[-tail_chars:]}"
```

> 这是第三层上下文保护：mini_compact（LLM 请求前）→ Compressor（超阈值时）→ _truncate_tool_result（存储时）

**4.4 `agent/tools/filesystem.py` — 读文件输出保护**

```
用户消息
    │
    ▼
[AgentLoop._process_message_inner]
    │
    ├── session.get_history()
    │   └── 返回最近 50 条 messages（现有逻辑不变）
    │
    ├── context.build_messages(summary=session.summary)
    │   ├── build_system_prompt()
    │   │   ├── _get_identity()          → System Context（时间、OS、workspace 路径等）
    │   │   ├── _load_bootstrap_files()  → AGENTS.md, SOUL.md, USER.md, TOOLS.md, IDENTITY.md
    │   │   ├── memory.get_memory_context() → ★ MEMORY.md + 当日笔记（已有机制，自动注入）
    │   │   └── skills                   → 技能摘要
    │   ├── summary → "## Conversation Summary" 段落（内容 = reconstructed_summary）
    │   ├── history（mini_compact 处理后）
    │   └── 当前 user message
    │
    ├── LLM 循环（tool calls / final response）
    │   └── agent 可调用 memory_recall 工具主动召回事件记忆
    │
    ├── 保存到 Session
    │
    └── _maybe_trigger_summarization(session, last_response)
        │
        └── 如果 prompt_tokens >= threshold:
            │
            └── compressor.fire_and_forget() ──→ 异步后台执行：
                │
                ├── [1] 在 messages_snapshot 上追加压缩指令（复用 KV cache）
                │       → LLM 返回 JSON：{qa_pairs, long_term_memories}
                │       → 模型自行跳过首尾不完整事件
                │
                ├── [2] qa_pairs → SQLite event_memories + event_keywords (aiosqlite)
                │
                ├── [3] long_term_memories → append 到 MEMORY.md
                │
                ├── [4] session.summary = reconstructed_summary（复用现有字段）
                │
                ├── [5] session.messages = 保留最近 3 轮
                │       └── 从 session.messages 的【当前状态】取尾部（防竞态）
                │       └── 裁剪点在 user message 边界，天然安全
                │
                └── [6] session_manager.save(session)
```

### 六、关键设计决策

| 决策点 | 方案 | 理由 |
|--------|------|------|
| 存储方案 | SQLite (aiosqlite) + 文件系统 | 轻量、异步安全、无外部依赖 |
| 替换 vs 并存 | 替换 Summarizer | 功能完全覆盖，避免两套系统并存 |
| 压缩方式 | 在当前 messages 上追加指令 | 复用 KV cache，效率高、准确性好 |
| 压缩范围 | 全部 messages 送 LLM | 模型自行跳过首尾不完整事件，避免硬编码切割丢失信息 |
| 裁剪方式 | 保留最近 3 轮（与 mini_compact 一致） | 固定规则，不依赖 LLM 判断，裁剪点天然安全 |
| 长期记忆存储 | 复用 MEMORY.md | 已有机制，`build_system_prompt` 自动注入 |
| summary 字段 | 复用 Session.summary，累积式更新 | previous + new QA，不覆盖，不丢失历史 |
| JSON 解析 | 带 fallback（code block 提取） | 防止 LLM 输出格式不规范 |
| 竞态防护 | 裁剪时从 session.messages 当前状态取尾部 | 压缩期间可能有新消息，不能从快照取 |

### 八、风险点与应对

| 风险 | 应对 |
|------|------|
| 压缩与主循环竞态 | 裁剪时从 session.messages 当前状态取尾部，不从快照取 |
| JSON 解析失败 | fallback 到正则提取 code block，仍失败则跳过本次压缩 |

### 十、沙箱集成测试 (`tests/test_memory_sandbox.py`)

#### 10.1 设计目标

使用**真实 session 数据**（非 mock）在**隔离沙箱**中回放压缩和召回流程，验证：
1. 压缩器是否正确提取 QA 对和 composite_key
2. 二次压缩是否正确去重（不重复提取 previous_summary 中已有事件）
3. 召回工具是否能通过 composite_key 正确检索
4. Summary 是否正确注入 system prompt

#### 10.2 沙箱隔离机制

```python
class SandboxSessionManager:
    """Mock SessionManager — save() 只写内存，不写磁盘。"""
    def __init__(self):
        self._saved: dict[str, Session] = {}
    def save(self, session: Session):
        self._saved[session.key] = session

def create_sandbox(provider, model, real_workspace) -> Sandbox:
    """创建隔离沙箱：
    - sandbox_root = tempfile.mkdtemp()
    - 复制 SOUL.md/TOOLS.md 等 bootstrap 文件到沙箱
    - compressor._db_path 指向沙箱内的 test_memory.db
    - MEMORY.md 在沙箱内初始化为空
    """
    sandbox_root = Path(tempfile.mkdtemp(prefix="nanobot_sandbox_"))
    sandbox_workspace = sandbox_root / "workspace"
    sandbox_workspace.mkdir()
    (sandbox_workspace / "memory").mkdir()
    
    # 复制 bootstrap 文件
    for filename in ["SOUL.md", "TOOLS.md", "USER.md", "AGENTS.md"]:
        src = real_workspace / filename
        if src.exists():
            shutil.copy2(src, sandbox_workspace / filename)
    
    # 初始化空 MEMORY.md
    (sandbox_workspace / "memory" / "MEMORY.md").write_text(
        "# Long-term Memory\n\n(sandbox — empty)\n"
    )
    
    compressor = Compressor(provider=provider, model=model, workspace=sandbox_workspace)
    compressor._db_path = sandbox_root / "test_memory.db"  # 隔离 SQLite
    
    recall_tool = MemoryRecallTool(compressor=compressor, provider=provider)
    
    return Sandbox(root=sandbox_root, workspace=sandbox_workspace, ...)
```

#### 10.3 三阶段测试流程

```
Phase 1 — 首次压缩
├── 输入：session 的前 N 条消息，无 previous_summary
├── 调用 LLM 压缩 → 解析 JSON → 存入沙箱 SQLite
└── 验证：
    ├── QA 对数量 > 0
    ├── System prompt 内容未泄漏到 QA 对中
    └── 所有 QA 对都有有效的 composite_key

Phase 2 — 二次压缩
├── 输入：session 的后续消息 + Phase 1 的 reconstructed_summary
├── 调用 LLM 压缩 → 解析 JSON → 存入沙箱 SQLite
└── 验证：
    ├── 新 QA 对来自第二批消息
    └── 未重复提取 Phase 1 summary 中已有的事件（去重有效）

Phase 3 — 召回测试
├── T1: 逐个 QA 对的 composite_key 召回
│   └── 使用每个 QA 的 composite_key 调用 search_by_composite_keys()
│   └── 验证召回率 ≥ 80%
├── T2: 多 composite_key 召回
│   └── 同时查询多个 key，验证返回多个事件
├── T3: 不存在的 composite_key
│   └── 验证返回空结果
├── T4: MemoryRecallTool 端到端测试
│   └── 创建 mock session → set_session() → execute(query=...)
│   └── 验证能召回匹配的事件
└── T5: Summary 注入 system prompt 验证
    └── build_messages(summary=...) → 验证 system prompt 包含 "Conversation Summary"
```

#### 10.4 运行方式

```bash
# 完整测试（Phase 1 + 2 + 3）
python3 tests/test_memory_sandbox.py --session sessions/dingtalk_xxx.jsonl

# 指定切分点（前 30 条 → Phase 1，其余 → Phase 2）

# 指定压缩模型
python3 tests/test_memory_sandbox.py --session sessions/dingtalk_xxx.jsonl --model qwen-max

# 只运行 Phase 3 召回测试

```

#### 10.5 关键设计决策

| 决策点 | 方案 | 理由 |
|--------|------|------|
| 数据来源 | 真实 session .jsonl | 避免 mock 数据无法暴露真实问题 |
| 隔离方式 | tmpdir + SQLite path 覆盖 | 零写入真实环境，测试后自动清理 |
