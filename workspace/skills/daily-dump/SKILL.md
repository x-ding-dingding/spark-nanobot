# Daily Dump 自动记录

你内置了一个自动工作日志记录模块，会在对话中自动识别并记录项目相关的内容到 `{WORK_DIR}/00_inbox/daily_dump.md` 文件中。

## 触发条件

当用户在对话中提到以下内容时，**自动、静默地**记录到 daily_dump.md：

- 提到具体的项目名称（如"XX 项目"、"XX 系统"）
- 描述项目的想法、思考、设计方案
- 报告项目的进展、完成的工作
- 讨论遇到的问题、解决方案
- 分享实现的细节、技术选型

**不要**在以下场景记录：
- 纯闲聊、问候
- 询问天气、时间等无关内容
- 没有过多信息量的内容
- 重复内容

## 记录规则

### 1. 识别项目

从用户的消息中提取项目名称，例如：
- "nanobot 项目"、"nanobot"
- "XX 系统"、"XX 平台"
- "XX 功能"、"XX 模块"

### 2. 提取内容

记录以下类型的内容：
- **想法/思考**：用户提出的新想法、设计思路
- **进展/完成**：已完成的工作、实现的功能
- **问题/解决**：遇到的问题及解决方案
- **细节**：实现过程中的技术细节、关键决策

### 3. 追加格式

使用以下 Markdown 格式追加到文件末尾：

```markdown
# [项目名称] [内容类型]


[记录内容]

---
```

**示例**：

```markdown
# nanobot 配置迁移

完成了配置文件从 ~/.nanobot/ 迁移到项目根目录的改造：
- 修改了 helpers.py 的 get_data_path() 逻辑
- 更新了所有硬编码的路径引用
- 迁移了 config.json、cron/jobs.json、sessions/ 等文件
- 更新了 .gitignore 排除敏感数据

---

# nanobot CBT 定时任务优化


调整了 CBT cron job 的行为：
- 默认改为轻松的进度确认，不直接进入 CBT 模式
- 只有当用户表达抗拒或不知道怎么开始时才切换到 CBT
- 修复了时区转换 bug，cron 任务现在能在正确的北京时间触发

---
```

### 4. 内容提取原则

- **保持原汁原味**：尽量使用用户的原话，不要过度加工
- **捕捉关键信息**：思考过程、决策理由、实现细节
- **简洁但完整**：不需要长篇大论，但要包含足够的上下文
- **不需要体系化**：按时间顺序追加即可，不需要分类整理（整理由 iflow-organizer 等其他 skill 完成）

## 执行流程

### 步骤 1：识别触发

在每次用户消息中，判断是否包含项目相关内容。如果包含，进入步骤 2。

### 步骤 2：提取信息

从用户消息和你的回复中提取：
- 项目名称
- 内容类型（想法/进展/问题/细节）
- 核心内容（2-5 句话概括）

### 步骤 3：静默记录

使用 `write_file` 工具（`append=true`）追加到 `{WORK_DIR}/00_inbox/daily_dump.md`：

```python
# 伪代码示例
content = f"""# {project_name} {content_type}

{extracted_content}

---

"""

write_file(
    path="{WORK_DIR}/00_inbox/daily_dump.md",
    content=content,
    append=True
)
```

**重要**：
- 必须设置 `append=True`，否则会覆盖文件原有内容
- 记录操作应该**静默执行**，不要告诉用户"我已经记录到 daily_dump.md"
- 如果文件不存在，自动创建
- 每次追加后自动添加 `---` 分隔符

### 步骤 4：继续对话

记录完成后，正常回复用户的问题，不要提及记录操作。

## 特殊场景

### 同一项目多次提及

如果用户在同一天多次提到同一个项目，**每次都创建新的条目**，不要合并：

```markdown
# nanobot 想法

想给 nanobot 加一个自动记录功能...

---

# nanobot 进展

实现了 daily-dump skill，可以自动记录项目内容了...

---
```

### 多个项目同时讨论

如果用户在一条消息中提到多个项目，分别为每个项目创建条目。

### 内容过长

如果提取的内容超过 10 行，适当精简，保留最关键的信息。

## 示例对话

**用户**：我在想 nanobot 的配置文件放在 ~/.nanobot/ 不太方便管理，能不能迁移到项目目录下？

**你的操作**：
1. 识别项目：nanobot
2. 内容类型：想法
3. 静默记录到 daily_dump.md：
   ```markdown
   # nanobot 配置迁移想法
      
   希望将配置文件从 ~/.nanobot/ 迁移到项目根目录，方便版本控制和管理。
   
   ---
   ```
4. 正常回复用户的问题

**用户**：刚完成了 nanobot 的配置迁移，修改了 helpers.py 和 loader.py 的路径逻辑。

**你的操作**：
1. 识别项目：nanobot
2. 内容类型：进展
3. 静默记录：
   ```markdown
   # nanobot 配置迁移完成
      
   完成配置文件迁移：
   - 修改 helpers.py 的 get_data_path() 改为项目根目录
   - 更新 loader.py 的 get_config_path()
   - 迁移 config.json、cron/、sessions/ 到项目目录
   
   ---
   ```
4. 回复："太好了！配置迁移完成后记得重启 nanobot 让新路径生效。"

## 注意事项

1. **静默记录**：不要告诉用户你在记录，保持对话自然流畅
2. **及时记录**：在回复用户之前就完成记录操作
3. **不要过度记录**：只记录有价值的项目相关内容，避免记录琐碎对话
4. **保持简洁**：每条记录 2-5 句话即可，不要写成长文
